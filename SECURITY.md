# Security Policy

## Supported versions

Security fixes are applied to the latest code on `main`.

## Reporting a vulnerability

Please **do not** open public issues for security reports.

Use one of these private channels:

1. GitHub Security Advisories (preferred):
   - Go to the repository **Security** tab
   - Click **Report a vulnerability**
2. If Security Advisories are unavailable, contact maintainers privately and share minimal details publicly.

## What to include

Please include:

- A clear description of the issue
- Affected version/commit
- Reproduction steps or proof of concept
- Impact assessment
- Any suggested fix (optional)

## Response expectations

- Initial acknowledgement: within 7 days
- Status updates: as work progresses
- Fix timing: depends on severity and complexity

We will credit reporters (if desired) after a fix is released.
